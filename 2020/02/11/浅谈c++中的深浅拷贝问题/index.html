浅谈c++中的深浅拷贝问题

1.为什么要引入深拷贝？

浅拷贝模式下面的赋值方式：逐字赋值

这样的话在析构时候p2由于先出栈，会先把堆中的内存释放掉，

然而p1再出栈时候，会再次把堆中的内存释放掉，这就造成了堆中内存的重复释放。

深拷贝模式：自己写一个拷贝构造函数，使得p2赋值时候指向别的内存，

从而使得堆区内存不会被重复释放

小结代码模式：浅拷贝：m_height=p.m_height;

​ 深拷贝：m_height=new int (*p.m_height);

//总结：如果属性有在堆区开辟的，一定要自己开辟空间，防止浅拷贝带来的问题
